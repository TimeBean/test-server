# Основная ветка репозитория

Данная ветка предназначена для хранения конфигурации производственного сервера. Предполагается, что она развёрнута на реальном сервере и в идеале должна обновляться исключительно в рамках глобальных обновлений.

---

## Философия

Все файлы, включаемые в репозиторий, должны удовлетворять двум критериям: **необходимость** и **невоспроизводимость**.

---

## Методология

### Общие принципы

При добавлении или изменении файла необходимо последовательно ответить на три вопроса:

1. **Необходим ли данный файл (или вносимое изменение)?** Если ответ отрицательный — файл (изменение) в репозиторий не включается.
2. **Может ли данный файл быть создан непосредственно на сервере?** Если да — файл в репозиторий не включается.
3. **Является ли файл воспроизводимым?** Если файл может быть сгенерирован автоматически (например, при запуске сервера) — вносить его в репозиторий не требуется.

### Частные замечания

1. Ядро сервера не хранится в репозитории; вместо этого хранится исключительно ссылка на соответствующий `.jar`-файл.
2. Вообще говоря, игровые миры не хранятся в репозитории. Вместо них рекомендуется использовать схемы FAWE.
3. Файлы, генерируемые автоматически и не содержащие на данный момент необходимых невоспроизводимых изменений, допускается включать в репозиторий при условии, что внесение таких изменений является обязательным и запланированным. Следует, однако, учитывать, что подобный подход может порождать технический долг и потому **не рекомендуется**.

---

## Схемы FAWE

Если некоторая структура является невоспроизводимой, её надлежит хранить в виде схемы FAWE.

Схемы должны именоваться по следующему шаблону:

```
{название_структуры}-{мир_установки}-{координаты_установки}.schem
```
